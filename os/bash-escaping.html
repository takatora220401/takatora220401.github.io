BashのEscapeについて


1. Escapeとは

特別に扱われる文字(例えば、$や?)から特別な意味を消すことですね。


2. エスケープ方法

  • \ (backslash) - 次に続く一文字を文字通りとします。
  • '' (single quotation) - すべての文字を文字通りの意味にします。''の中に'が入らないように注意が必要です。
  • "" (double quotation) - $, `,\を除くすべての文字を文字通りの意味にします。

参考：

  • 3.1.2.1 Escape Character
  • 3.1.2.2 Single Quotes
  • 3.1.2.3 Double Quotes

3. 変数の中の特殊文字

次のコードを見てください。

$ test1="\$PWD"
$ echo $test1
$PWD

変数testの中身($PWD)をechoした時、$PWDは展開されないですよね。つまり、変数の中身はエスケープされていなくても展開されないみたいですね。Bash script won't interpret variable within a variableでも同じことが議論されてます。


おまけ

少し脱線しますが、Bashが認識するEscape Sequence(\nや\w)について、\n以外は登場シーンがPS1の定義に限定されると考えてもいいともいます。3.1.2.1 Escape Characterにも\nの記載しかありませんしね。

A non-quoted backslash ‘\’ is the Bash escape character. It preserves the literal value of the next character that follows, removing any special meaning it has, with the exception of newline. If a \newline pair appears, and the backslash itself is not quoted, the \newline is treated as a line continuation (that is, it is removed from the input stream and effectively ignored).
