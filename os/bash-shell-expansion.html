Bashの基本の1つ、シェル展開についてまとめます。


1. Bashのシェル展開とは

placeholderに値を代入する機能です。構文の形式によって代入される値が異なります。例えば、ブレース展開{1..4}には1 2 3 4が代入されます。その他にも次のようなシェル展開があります。

  • ~ (tilde)展開
  • パラメータ展開
  • 算術式展開
  • プロセス置換
  • コマンド置換
  • グロビング

2. 各展開の動作

~ (tilde)展開

~の部分に$HOMEの値をいれます。次の例ではユーザーのホームディレクトリに移動します。

$ cd ~

パラメータ展開

パラメータの位置にその中身をいれます。Bashにおけるパラメータは値を格納する箱のようなものです。実は変数はパラメータの1つです。パラメータにはその他にも、位置パラメータや特殊パラメータ、配列の要素などがあります。

#変数の場合
$ echo $PWD
/home/thxxxxxo

#位置パラメータの場合
$ cat positional-parameter-test.sh 
#!/bin/bash
echo script name: $0 
echo 1st argument:$1
$ ./positional-parameter-test.sh arg1
script name: ./positional-parameter-test.sh
1st argument:arg1

#特殊パラメータの場合
$ cat special-parameter-test.sh
#!/bin/bash
echo "$@"
counter=1
for e in "$@"; do
  echo  element$counter: $e
  ((counter+=1))
done
$ ./special-parameter-test.sh "arg1 arg2" arg3
arg1 arg2 arg3
element1: arg1 arg2
element2: arg3

算術式展開

計算の値をいれます。


$ echo $((2026%5))
1

プロセス置換

コマンド置換とかなり似てますが、プロセス置換はコマンドの実行結果をファイルとして渡します。

$ echo <(ls)
/dev/fd/63

コマンド置換

プロセス置換がファイルとして実行結果を渡すのに対して、コマンド置換ではそのまま結果を渡します。

$ echo $(ls)
BashCourse DjangoProject Github_Actions Go_basics NodejsCourse pythonForHacker

グロビング

コマンドの対象となるファイルを楽に書くためのショートカットを提供するものって感じです。

$ # ファイル一覧
ls
01.txt  03.txt  05.txt  07.txt  09.txt  11.txt  13.txt  15.txt  17.txt  19.txt
02.txt  04.txt  06.txt  08.txt  10.txt  12.txt  14.txt  16.txt  18.txt  20.txt 

# []は中でフィルターを指定
$ ls [0-1][7-9].txt
07.txt  08.txt  09.txt  17.txt  18.txt  19.txt

# ?は任意の文字1つに合致
$ ls 0?.txt
01.txt  02.txt  03.txt  04.txt  05.txt  06.txt  07.txt  08.txt  09.txt

# *は任意の長さの任意の文字に合致
$ ls *.txt
01.txt  03.txt  05.txt  07.txt  09.txt  11.txt  13.txt  15.txt  17.txt  19.txt
02.txt  04.txt  06.txt  08.txt  10.txt  12.txt  14.txt  16.txt  18.txt  20.txt
