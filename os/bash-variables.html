Bash変数とは

筑波大学がBash変数について分かりやすくまとめてくださってますね。

参考：シェル変数、環境変数、シェル・スクリプト


大事なのはこのあたりでしょうか。

変数　　　- 何かを覚えるためのもの。メモリを抽象化して名前をつけたもの 変数名　　- 変数を区別するための名前 変数の値　- 変数が覚えているもの シェルでは、2種類の変数を使える ・シェル変数 - １つのシェル内でのみ有効 ・環境変数 - 実行されるプログラム(子プロセス)に引き継がれる

シェル変数と環境変数を並列に捉えるより、シェル変数の中でも子シェルに引き継がれるものを特別に環境変数と呼び、そうでないものをローカル変数と考えたほうが実態に近い気がします。そして、シェル関数を環境変数に昇格させる方法の1つがexportコマンドというわけですね。

参考：

  • 3.7.4 Environment
  • Exporting shell variables (export shell command)

環境変数

  • $PATH - 実行ファイルを探すフォルダのリスト
  • $HOME - ユーザーのホームディレクトリ
  • $USER - ログイン中のユーザー
  • $PWD - ワーキングディレクトリ（ファイルシステム上の現在地）
